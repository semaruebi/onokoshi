# 🎯 狩り残し確認チェッカー

精鋭狩りRTA中に**倒しきれなかった精鋭エネミー（狩り残し）**を効率的にチェック・記録し、次回のRTAのための分析材料を収集するためのWebアプリケーションです。

## ✨ 機能

### 🗺️ ルート情報の登録・管理
- 精鋭グループ名とその数を一行ずつ入力してルートを登録
- ルート名を付けて保存・呼び出しが可能
- 登録済みルートの一覧表示と削除

### ✅ 狩り残しチェック機能
- 登録されたルートリストを順番にチェック
- 精鋭グループごとに「狩り残しがいたかどうか」を記録
- 狩り残しがあった場合、メモを入力可能
- プログレスバーで進捗を視覚的に表示
- 前へ/次へボタンでスムーズに移動

### 📈 履歴・分析機能
- チェック結果を日付とルート名と一緒に保存
- 過去のチェック履歴を一覧で閲覧
- 精鋭グループ別の狩り残し率を統計表示
- メモの内容を一覧表示して分析に活用

## 🚀 セットアップ

### 必要な環境
- Node.js 18以上
- npm または yarn

### インストール

```bash
npm install
```

### 開発サーバーの起動

```bash
npm run dev
```

ブラウザで `http://localhost:5173` を開いてください。

### ビルド

```bash
npm run build
```

ビルドされたファイルは `dist` ディレクトリに出力されます。

## 💾 データ保存

すべてのデータはブラウザのローカルストレージに保存されます。サーバー通信は一切ありません。

- ルート情報: `hunt-checker-routes`
- チェック履歴: `hunt-checker-histories`

## 📝 使用方法

1. **ルートの登録**
   - 「新しいルートを登録」フォームでルート名と精鋭グループリストを入力
   - 形式: `精鋭グループ名 数`（例: `かつヴァナ 5`）

2. **チェックの開始**
   - 登録済みルート一覧からルートを選択
   - 各グループごとに「狩り残しがいた」をチェック
   - 必要に応じてメモを入力
   - 「次へ」ボタンで次のグループへ移動
   - 全て完了したら「完了」ボタンをクリック

3. **履歴の確認**
   - 「履歴」タブから過去のチェック結果を確認

4. **統計の確認**
   - 「統計」タブから精鋭グループ別の狩り残し率を確認
